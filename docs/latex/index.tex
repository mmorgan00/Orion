Welcome to the documentation for the Orion Engine project! This project is a solo game engine written in C, designed to support multiple graphics APIs, starting with Vulkan. The goal of this engine is to provide primarily a hands on learning experience for myself, and serve as a \textquotesingle{}proving ground\textquotesingle{} for new rendering techniques and continually futhering my knowledge.\hypertarget{index_autotoc_md1}{}\doxysection{\texorpdfstring{Current Features}{Current Features}}\label{index_autotoc_md1}
The game engine aims to support a wide range of features, for now the following are available\+:


\begin{DoxyItemize}
\item {\bfseries{Graphics APIs}}\+: Initial support for Vulkan, with structure in place to enable multiple graphics APIs without changing game API
\item {\bfseries{Shader Mangement}}\+: Support for GLSL shader compilation and linkage, with plans to support HLSL as well
\item {\bfseries{Input Handling}}\+: Cross platform input handling, with plans and structure in place to support gamepads and custom controllers
\item {\bfseries{Event System}}\+: Configurable system with event registration for a variety of events both user level and system level
\item {\bfseries{Memory Allocator}}\+: Cross platform memory allocator with standard API exposed to both game and engine
\item {\bfseries{Math Library}}\+: Engine provided math library featuring 2D and 3D matrices, Quaternion structures, and trig functions
\end{DoxyItemize}\hypertarget{index_autotoc_md2}{}\doxysection{\texorpdfstring{Planned Features}{Planned Features}}\label{index_autotoc_md2}
In addition to existing features, there a number of modules that are planned for development\+:


\begin{DoxyItemize}
\item {\bfseries{Material Management}}\+: Expose a Material api, including textures, shader maps (normal, displacement, etc) and PBR support
\item {\bfseries{Scene Graph}}\+: A hierarchical structure to manage and render game objects.
\item {\bfseries{Physics Engine}}\+: Custom physics engine to support various simulations
\item {\bfseries{Audio Engine}}\+: Support for 3D sound and sound effects.
\item {\bfseries{Scripting}}\+: Support for scripting languages to allow for flexible game logic.
\end{DoxyItemize}\hypertarget{index_autotoc_md3}{}\doxysection{\texorpdfstring{Current Development Status}{Current Development Status}}\label{index_autotoc_md3}
At the moment, the primary focus is a solid foundation of engine structure and familiarity with Vulkan. The latest version supports the \textquotesingle{}Current Features\textquotesingle{} as well as the following in-\/development features\+:


\begin{DoxyItemize}
\item {\bfseries{Texture Loading}}\+: Implementing functionality to load textures from various formats.
\item {\bfseries{Texture Mapping}}\+: Developing methods to map textures onto 3D models.
\item {\bfseries{Shader Integration}}\+: Linking texture samplers into shaders for object rendering and render-\/to-\/texture support
\item {\bfseries{Unit Tests}}\+: Proper support for unit testing of various features, such as memory allocator, resource loading, and others
\end{DoxyItemize}\hypertarget{index_autotoc_md4}{}\doxysection{\texorpdfstring{Getting Started}{Getting Started}}\label{index_autotoc_md4}
